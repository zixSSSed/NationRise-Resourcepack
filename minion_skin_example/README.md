# Скин миньона — как сделать (шаблон)

Миньон в мире = маленький **армор-стенд**, у которого в слоте **головы** лежит предмет-носитель
с **CustomModelData**. Ресурспак мапит этот номер на твою 3D-модель. Меняешь номер → меняется «скин».

## Файлы примера
- `nr_minion_demo.png` — текстура (16×16, заглушка-«лицо»).
- `nr_minion_demo.json` — 3D-модель (тело+голова+2 руки). Открывается в Blockbench.

## Как сделать СВОЙ скин (реальный путь — Blockbench)
1. Скачай **Blockbench** (бесплатно, blockbench.net).
2. New → **Generic Model** (это и есть item-модель Minecraft).
3. Нарисуй миньона из кубиков (Cube), наложи текстуру, сделай UV-развёртку (Blockbench сам).
   - Делай его так, чтобы «ноги» были у низа, а точка крепления (0,0,0 модели) — там, где будет голова стенда.
4. File → **Export → Export Block/Item Model** → получишь `.json` + текстуру `.png`.
   (Можешь открыть мой `nr_minion_demo.json` как образец структуры — File → Import.)
5. Скинь мне эти 2 файла (или положи в `pack/assets/minecraft/models/item/` и `.../textures/item/`).

## Что делаю Я (обвязка)
- Кладу твою модель в `pack/assets/minecraft/models/item/nr_minion_<имя>.json`
  и текстуру в `.../textures/item/nr_minion_<имя>.png`.
- Регистрирую её на **CustomModelData** (например 70001, 70002, …) в `items/<носитель>.json`
  (range_dispatch — как у building-иконок).
- В меню миньона делаю слот **«Скин»**: кладёшь предмет-триггер (блок или скин-токен) →
  миньону ставится этот CMD → армор-стенд показывает твою модель. Скин хранится в PDC (переносится).
- Тонкая настройка `display.head` (translation/scale) — подгоняем в игре, чтобы стоял ровно на земле.

## Одежда vs скин (как это устроено сейчас — v0.1.46)
- **Без скина** миньон = армор-стенд, который теперь **одет**: профильные руды носят настоящую броню
  (железо/золото/алмаз/незерит), остальные — крашеную кожу в цвет ресурса (лесоруб — зелёный,
  фермер — соломенный, землекоп — хаки, шахтёр — серый, редстоун — красный, лазурит — синий и т.д.).
  Голова — обычная голова игрока. Это убирает «голый стенд» сразу, без ресурспака.
- **Со скином** (CustomModelData на голове) модель — **цельный персонаж**, поэтому плагин
  **снимает броню с тела**, чтобы твоя модель не клиппилась. Весь вид = твоя модель.

## Хоумлендер (готовый пример скина)
- `nr_minion_homelander.png` — текстура-лицо (блонд, голубые глаза, бело-красно-золотой костюм).
- `nr_minion_homelander.json` — стартовая цельная модель (ноги/тело/2 руки/голова). Дорабатываешь UV в Blockbench.
- Превью: `_homelander_preview.png`.

## ГОТОВО и подключено (v0.1.47)
**Носитель скина = PAPER + CustomModelData** (НЕ player_head — пак мапит CMD только на бумагу!).
Это и была причина, почему раньше «снималась одежда, а скин не появлялся».

Уже в паке (`NationRise-Resourcepack.zip`):
- модель `pack/.../models/item/nr_minion_homelander.json` (3D-голова, display.head),
- текстура `pack/.../textures/item/nr_minion_homelander.png`,
- запись **CMD 70001 → nr_minion_homelander** в `items/paper.json`.
→ **Перезалей пак на сервер** (игрокам прилетит обновление).

Команды (касса сайта/админ, `townlands.admin`):
- `/nrminion skin Homelander` — надеть скин на ближайшего миньона (≤6 блоков). `/nrminion skin off` — снять.
- `/nrminion skingive <ник> Homelander` — выдать **торгуемый токен** скина (для доната/кассы).
- `/nrminion skins` — список доступных скинов.

Игроки (без прав): **шифт-клик токена** скина в меню миньона (слот 25 «🎭 Скин») — надевает; клик по слоту — снимает и возвращает токен. Скины торгуются между игроками (+экономика).

Скин хранится в PDC миньона и **переносится при сборке** (/собрать → поставить).

### Добавить НОВЫЙ скин (рецепт для меня)
1. Текстуру `nr_minion_<id>.png` → `pack/.../textures/item/`, модель `nr_minion_<id>.json` → `pack/.../models/item/`.
2. Запись `threshold: 700XX → minecraft:item/nr_minion_<id>` в `items/paper.json`.
3. В коде `MinionSkins.reg("<id>", "Имя", 700XX)`.
4. `py pack_zip.py`, собрать плагин, перезалить пак.

> Подгонка `display.head` (translation/scale) — по скрину из игры (одна правка чисел в модели).

## Самое важное в модели
- **`elements`** — кубики (from/to в координатах 0..16, можно выходить за пределы для крупных моделей).
- **`textures`** + **UV** на гранях — какая часть текстуры на какой грани.
- **`display.head`** — как модель крепится к голове стенда (тут главная подгонка размера/высоты).
- На каждый скин — отдельный CMD-номер. Их и переключаем в слоте «Скин».
